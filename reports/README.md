# Reports

## Scope

- Root-level reports for project policy, audit, and cross-module decisions.
- Module reports live under `modules/<NN>_<name>/reports/`.

## Naming policy

- Use UTC timestamp naming: `YYYYMMDD_HHMMSS_<topic>.md`.
- Topic should be short and descriptive, using lowercase and underscores.

## Creation policy

- Add a report whenever a cross-module decision or milestone checkpoint is reached.
- Update this index when a report is added.
- Report kinds should be concise (`audit`, `governance`, `decision`, `validation`, `handoff`, etc.).
